🎯 このツールについて

        
3つの攻撃手法（ブルートフォース・辞書攻撃・リバースブルートフォース）の違いを視覚的に理解できる教育用デモツールです。

        
        
📖 3つの攻撃モード

        
1. ブルートフォース（総当たり攻撃）

        
    
          
  • 手法： 8文字の全組み合わせを順次試行（aaaaaaaa, aaaaaaab, ...）
    • 
          
  • 特徴： 1人のユーザーに集中攻撃、ロックアウトが発生しやすい
    • 
          
  • デモ設定： ロックアウト無効時は約21秒で成功
    • 
        

        
        
2. 辞書攻撃

        
    
          
  • 手法： よくあるパスワード（123456, password等）から順番に試行
    • 
          
  • 特徴： 人間が使いがちなパスワードに特化、効率的
    • 
          
  • デモ設定： 辞書サイズ可変、デフォルト20件で3回目にヒット
    • 
        

        
        
3. リバースブルートフォース（パスワードスプレー）

        
    
          
  • 手法： 1つのパスワードを全ユーザーに試行、次のパスワードで再度全ユーザーに試行
    • 
          
  • 特徴： 各アカウントの失敗回数を分散、ロックアウト回避
    • 
          
  • 危険性： アカウント単位制限では防げない
    • 
        


        
🛡️ 防御の要点

        
    
          
  • 多層防御： IP・デバイス・サブネット・グローバルレート制限
    • 
          
  • パスワードポリシー： 共通・漏えいPW拒否、複雑性要求
    • 
          
  • 多要素認証： リスク検知時のMFA強制
    • 
          
  • 横断相関検知： 同一PW多アカウント試行の監視
    • 
          
  • 監査・可視化： リアルタイム異常検知とアラート
    • 
        


        
💡 使い方

        
    
          
  1. 環境設定： ユーザー数・弱パスワード比率・ロックアウト設定を調整
    2. 
          
  2. 環境生成： 「環境を生成」で仮想環境を作成
    3. 
          
  3. 攻撃比較： 各タブで異なる攻撃手法を実行
    4. 
          
  4. 効果確認： KPI・ログで防御効果の違いを観察
    5. 
          
  5. 設定変更： ロックアウト有無を切り替えて効果を比較
    6. 
        


        
🔍 観察ポイント

        
    
          
  • ロックアウト効果： 有効/無効での成功率・時間の違い
    • 
          
  • 攻撃手法比較： 各手法の特性とロックアウト発生パターン
    • 
          
  • 生成vs送信： ブルートフォースでの実際の通信量
    • 
          
  • 防御限界： アカウント単位制限の限界とその対策
    •
